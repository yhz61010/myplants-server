# MyPlants Server - AI 代理指令

## 项目概述
这是一个植物知识和栽培分享平台的服务器端应用程序。用户可以学习植物知识、上传和查看植物知识、管理个人资料，并通过类似时间线的社交动态分享栽培经验。

## 技术栈
- **语言**：Go
- **框架**：Gin（Web 框架）
- **数据库**：SQLite（配合 GORM）
- **目的**：植物相关社交平台的 RESTful API 服务器

## 主要功能
- 用户资料管理（显示和修改）
- 植物知识上传和显示系统
- 社交分享功能（类似动态/时间线），用于用户栽培日志和经验分享
- 植物信息管理（包括科学分类、描述、图片等）

## 开发约定
- 遵循标准的 Go 命名约定和项目结构
- 使用 Gin 进行 HTTP 路由和中间件
- 实现 RESTful API 端点
- 适当处理用户认证和授权
- 确保正确的错误处理和日志记录

## 构建和测试命令
- 初始化依赖：`go mod tidy`
- 构建应用程序：`go build`
- 运行测试：`go test ./...`
- 启动开发服务器：`go run main.go`（假设 main.go 是入口点）

## 关键目录/文件
- `main.go`：应用程序入口点
- `handlers/`：HTTP 请求处理器
- `models/`：数据模型和结构体（包括 Diary 和 Plant）
- `middleware/`：Gin 中间件组件
- `routes/`：API 路由定义
- `docs/db/`：数据库设计文档

## 常见模式
- 使用 Gin 的上下文进行请求/响应处理
- 为 API 响应实现正确的 JSON 序列化
- 使用 Go 标准库进行核心功能
- 遵循 REST 约定进行端点设计

## 潜在陷阱

- 为 API 端点实现速率限制
